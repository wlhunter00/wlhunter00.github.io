Practicum

  • May 2020

  • Co-Founder

  • Personal Startup

Problem

Students can only work in an agile environment through an internship, but COVID-19 canceled many of these opportunities. Without the formal structure of a company, it is difficult for CS students to work on a meaningful project.

Product

Practicum is designed to introduce students to a community of teams that operate as startups. The projects are sourced from entrepreneurs that need help developing an MVP. Teams have mentors that guide the students through the program.

Process

  • Developed a recruitment pipeline, which generated 300+ applications and 15+ clients in two weeks. Wrote out best practices for development.
  • Led a team of 8 partners to build the curriculum. The primary KPI was conversion rate between interview rounds, which averaged 88%.
  • Vetted potential clients to ensure quality, maintaining a 33% acceptance rate. Composed a feature list for each product for efficent onboarding.

RedditSense

  • January 2020

  • Risk Manager

  • Alpha Delta Phi

Problem

The early stages of my organization’s recruitment cycle are very reliant on qualitative observations about potential candidates. With a large number of students to consider, there was no quantitative measure of fit for the organization.

Product

The Recruitment Reddit Bot is a python bot that gathers data regarding comments members have left about candidates in a private subreddit. It assigns sentiment to the messages and then calculates the traction a candidate has within the organization.

Process

  • Utilized Reddit Praw Python API to parse Reddit 1200+ comments using Regex. Parsed keywords to determine the sentiment of feedback.
  • The bot used multiple metrics such as upvotes, comment sentiment, and engagement to calculate an excitement score for a candidate.
  • Automatically populated a Google Sheets document so the recruitment team could track the statistics generated.
Python PRAW API Google API Data Science

SpikeFinder

  • December 2019

  • Personal Project

  • WashU

Problem

At WashU, there is a large portion of the community that loves to play a game of Spikeball, and but there is no centralized system to find a group. The target audience is first-year students who are new to campus and are looking for people to meet.

Product

SpikeFinder is a group finder platform to help people passionate about the sport find like minded individuals. The product brings people together by allowing them to post their availability and group size and enables others to join the session to start playing.

Process

  • Developed frontend in React to ensure quick load times. Implemented React Hooks to increase scalability throughout the codebase.
  • Built backend Node.js REST API with Express, implementing authentication tokens to guard user data through OAuth.
  • Stored data in MongoDB NoSQL database and implemented Mongoose to fetch data using SQL-queries.
React Node.js Express MongoDB

Opportunity Hunter

  • June 2019

  • Software Development Intern

  • Alvarez & Marsal

Problem

Most of the clients that the Data Forensics works with are sourced by upper-management. However, there is a large market of public sector data opportunities that the firm wasn’t taping into, and the team was interested in this potential revenue stream.

Product

The Opportunity Hunter is a python tool that automatically scrapes RFP’s (Request For Proposal) posted by local and national governments regarding data forensics. This information is then parsed and presented to the team in a formal daily report.

Process

  • Scraped information from sites using Python libraries BeautifulSoup and Selenium to parse website’s source code, as well as API requests.
  • Wrote SQL queries to clean and store gathered data in an MS SQL Server database. Parsed the information to determine relevance to the team.
  • Built Pandas DataFrames to export data to an Excel spreadsheet to provide 20,000+ potential business opportunities in an emailed report.
Python MS SQL Selenium Beautiful Soup
